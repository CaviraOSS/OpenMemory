# Full GitHub Issues And PR Audit

## Scope
- Repository: `CaviraOSS/OpenMemory`.
- Date: 2026-05-19.
- Source: public GitHub listing pages plus each individual issue and pull request page fetched from GitHub HTML. The REST API was rate-limited and `gh` is not installed locally.
- Coverage: 42 real issues and 94 pull requests, including open and closed/merged items visible in GitHub public history.
- Machine-readable extraction is stored in `docs/github-issues-prs-full-audit.json`.

## Global Read
- The project history is dominated by mixed surfaces: MCP, dashboard/frontend, Docker/deploy, Python/JS packaging, SQLite/Postgres storage, embedding providers, source ingestion, and HSG scoring.
- The rewrite direction is correct: keep the active runtime narrow, durable `/v1`, Postgres-first, and rebuild integrations only through explicit durable boundaries.
- The most important user-visible risks to keep testing are Unicode/multilingual storage, tenant/project isolation, protocol cleanliness for future MCP, and simple install/start behavior.
